ELYSIUM

A digital table for Vampire: The Eternal Struggle

Version 1.7  ·  free  ·  no account  ·  everything runs in the browser

Host — 5 minutes, once

One person at the table runs the server. This is all of it:

  1. Install Node.js (the engine that runs the server): go to nodejs.org, download the LTS version, run the installer and click through with the defaults. Done in two minutes.
  2. Download the Elysium files from the project's GitHub page (the same place this guide lives) and put them in a folder of their own — at minimum the server elysium-server.js, the client elysium-vtes-bord.html (the game itself — one single file), and the launcher for your system: start-elysium.bat on Windows, or start-elysium.sh + start-elysium.command on macOS/Linux.
  3. Start the server:
    Windows — double-click start-elysium.bat.
    Want a server password, a tournament referee, or a single-room server? Double-click start-elysium-guided instead — it asks a few questions and starts with the right options. The plain launcher stays the zero-questions default. macOS — double-click start-elysium.command (the very first time: right-click → Open, since macOS is wary of new files; if it still refuses, open Terminal in the folder and run bash start-elysium.sh).
    Linux — in a terminal: bash start-elysium.sh (or once: chmod +x start-elysium.sh, then ./start-elysium.sh).
    A window opens and shows, among other things, a line starting with Network: — that is the address you share with the table. Leave it open while you play (Ctrl+C shuts down and saves the rooms).
  4. Create the room: open the address yourself in a browser, ☰ → Play online… → fill in a room name, a password and an optional match clock → Create room. Then click 📋 invite in the room bar and a ready-made invitation lands on your clipboard — paste it into Discord and send the password separately.

Important: never double-click the .js file itself — Windows will run it with the wrong engine and complain about an "Invalid character" (in your system language). On macOS/Linux a double-clicked .js usually just opens in a text editor. That's exactly why the launchers exist — they check for Node and start it the right way on every system.

Guest — 30 seconds

Got an address from your host? Then you're nearly done.

  1. Open the address in your browser — e.g. http://192.168.1.42:8123 or a Tailscale address. The game loads instantly; nothing is installed.
  2. Click ☰ → Play online… — the server address is already filled in. Enter your name, the room name and the password you were given.
  3. Join room. Import your deck (or take the demo deck), click Deal — and play.

Just want to watch? Tick Join as spectator and you'll see the whole table without being able to touch anything.

Prefer your own copy? You can also download the client file elysium-vtes-bord.html from the project's GitHub page and open it straight from your computer (just double-click it) — then ☰ → Play online… and type the host's address yourself. Your saved decks and downloaded card images then live with your file and survive between game nights, and a local images/ folder next to it works too.

Playing on a tablet? Elysium is touch-native: long-press anything for its menu · drag moves cards · pinch zooms, one finger on the felt pans · tap the ± buttons or a counter badge to adjust pool and blood · tap the Library/Crypt to draw. On an iPad, use Share → Add to Home Screen for a clean fullscreen table.

Internet — when the table doesn't share WiFi

The easiest road is Tailscale — a free private network between you.

  1. The host creates a free account at tailscale.com and installs the app.
  2. The host sends an invite link (Tailscale panel → Invite) to the players.
  3. Each player installs the app via the link and signs in. That's it — you're now on the same "LAN".
  4. The host starts Elysium as usual and shares their Tailscale address (starts with 100., shown in the Tailscale app) plus :8123.
  5. Guests open http://100.x.y.z:8123 in their browsers. Done — encrypted, and without opening a single router port.
Alternative — nothing for guests to install

Cloudflare Quick Tunnel gives you a public https address in seconds. Guests just open the link — no app, no account.

  1. Get cloudflared (once):
    Windows — download cloudflared-windows-amd64.exe from github.com/cloudflare/cloudflared/releases/latest, rename it to cloudflared.exe, and place it in your Elysium folder.
    macOS — brew install cloudflared (or the release binary above, saved in the folder as cloudflared + chmod +x cloudflared).
    Linux — the apt/yum repo at pkg.cloudflare.com, or the release binary saved in the folder as cloudflared + chmod +x cloudflared.
  2. Start the Elysium server as usual (the launcher for your system, per the Host section above).
  3. Start the tunnel: Windows — double-click start-cloudflare-tunnel.bat; macOS — double-click start-cloudflare-tunnel.command (same first-time right-click → Open as above); Linux — bash start-cloudflare-tunnel.sh. After a few seconds the tunnel address appears, printed in a big banner — and lands on your clipboard automatically (Windows/macOS; Linux too if xclip is installed).
  4. Paste the address to your players (Ctrl+V). They open it in any browser and connect as normal. Elysium automatically uses an encrypted connection when served over https.

The tunnel address is temporary and changes every time you start it. Closing the tunnel window disconnects guests, but the server and room stay alive — start the tunnel again, share the new address, and everyone reconnects automatically. Don't open a port on your router — that's the road we advise against.

Tip: set your computer to never sleep while hosting (Windows: Settings → System → Power; macOS: System Settings → Displays/Battery → prevent sleeping). The Elysium server survives sleep just fine, but the tunnel connection may not — if it drops, restart the tunnel and share the new address.

If something snags
"Invalid character, line 1" when starting the serverYou double-clicked the .js file. Use start-elysium.bat.
Works on the host machine but not for guestsWindows Firewall: Windows Security → Firewall & network protection → Allow an app → tick Node.js for private networks, and check the network profile is Private. macOS: click Allow when the firewall asks about incoming connections for node (or add it under System Settings → Network → Firewall).
The page won't even load for a guestGuest networks/mesh with "client isolation" block device-to-device traffic — switch to the main network, or use Tailscale.
"cloudflared.exe was not found" when starting the tunnelThe download from GitHub is called cloudflared-windows-amd64.exe (64-bit) or cloudflared-windows-386.exe (32-bit). Rename it to cloudflared.exe and place it next to start-cloudflare-tunnel.bat. Almost all modern PCs are 64-bit — pick amd64.
macOS: "… cannot be opened" for the .command fileGatekeeper being careful with a new file. Right-click it → Open → Open (needed once). Or run bash start-elysium.sh in Terminal.
"Permission denied" when running a .shEither prefix it: bash start-elysium.sh — or make it executable once: chmod +x start-elysium.sh.
"cloudflared: command not found" (macOS/Linux)brew install cloudflared on a Mac; on Linux the pkg.cloudflare.com repo — or save the release binary next to the script as ./cloudflared and chmod +x cloudflared.
"That name is taken"The name is currently signed in to the room. An offline seat is reclaimed with the same name + the room password.
